# 示例

## 示例 1：只生成招聘内容

### 输入

```markdown
岗位名称：Java后端工程师（AI Agent + IoT）

岗位职责：
1. 负责公司 AI Agent 平台与 IoT 平台的核心功能研发，构建稳定、高效的智能服务平台。
2. 独立完成相关模块或系统的架构设计、技术选型与编码实现，确保系统高可用、可扩展。
3. 持续对线上服务进行性能、稳定性与安全性优化，保障线上服务 SLA。

任职要求：
1. 掌握 Java 基础技术体系，熟练使用 SpringBoot、SpringCloud、Spring AI。
2. 熟悉 MySQL、Redis、RabbitMQ 等中间件原理，可根据需求合理运用。
3. 熟悉 Docker、K8s 容器化技术。
4. 熟练使用日志平台、链路监控平台。
5. 熟悉研发规范、敏捷开发模式。
6. 必须熟练使用 AI 编程工具。
```

### 输出示例

```markdown
## 任务目标
- 本次输出：岗位画像 + 筛选标准 + 结构化面试题

## 岗位画像
- 面向 AI Agent + IoT 平台的核心后端研发岗，重点是稳定性、扩展性与线上 SLA。
- 需要候选人既能做 Java 云原生后端，也能支撑复杂设备链路与智能体能力落地。

## 筛选标准
### 必须具备
- 3 年以上 Java 后端经验，具备 SpringBoot / SpringCloud 生产实践。
- 有 MySQL、Redis、RabbitMQ 的真实线上使用经验，能说清选型和治理方法。
- 有 Docker / K8s 使用经验，能配合完成服务部署与运维协作。
- 能使用日志与链路追踪平台排查线上问题。
- 熟练使用 AI 编程工具，并能举出提效案例。

### 可培养
- Spring AI 深层实践经验。
- AI Agent 编排或工具调用链路经验。

### 加分项
- IoT 设备接入、控制链路、协议处理经验。
- 做过高并发链路优化、SLA 保障、故障演练。

## 结构化输出
### 面试题
1. 你负责过的最复杂 AI Agent 或 IoT 后端模块是什么？请讲清核心链路、瓶颈和优化方式。
2. Redis 与 RabbitMQ 在高并发场景下如何配合实现削峰与最终一致？
3. 线上接口 P99 延迟突然升高时，你如何用日志和链路追踪快速定位问题？
4. 如果要给 Agent 工具调用链路增加容错与观测能力，你会怎么设计？

## 风险与待验证项
- 候选人是否真正做过 Agent 工具编排，需要通过项目深挖验证。

## 下一步建议
- 先安排 45 分钟技术一面，重点考察架构设计与线上故障处理。
```

## 示例 2：带候选人信息的评分结论

### 输入

```markdown
岗位名称：Java后端工程师（AI Agent + IoT）
岗位职责：同上
任职要求：同上

候选人信息：
- 简历摘要：6 年 Java 后端，做过 IoT 设备管理平台，最近一年参与 Agent 工作流平台开发
- 技术栈：SpringBoot、SpringCloud、MySQL、Redis、RabbitMQ、Docker、K8s、SkyWalking、ELK
- 项目经历：主导设备控制链路重构，P99 从 900ms 降到 280ms
- AI工具使用：日常使用 Cursor + Copilot，能展示提示词与测试生成实践
```

### 输出示例

```markdown
## 任务目标
- 本次输出：简历初筛 + 评分卡 + 录用建议

## 岗位画像
- 岗位要求候选人兼顾 Java 云原生工程能力、IoT 链路稳定性治理、Agent 平台协同开发经验。

## 候选人评分卡
- Java 与框架：22/25（Java 栈完整，Spring AI 深度仍需追问）
- 中间件：13/15（有生产实践，也有性能优化结果）
- 云原生：13/15（具备 Docker / K8s 落地经验）
- 可观测性：9/10（SkyWalking + ELK 使用清晰）
- 架构与工程能力：17/20（有链路重构与指标改善结果）
- AI 编程工具：13/15（持续使用，有可说明的提效方式）
- 总分：87/100

## 风险与待验证项
- Agent 工作流平台参与深度不明，需要确认是否主导过关键模块设计。
- Spring AI 的实战深度仍需通过系统设计题验证。

## 下一步建议
- 建议进入下一轮。
- 安排 60 分钟系统设计面，重点考察 Agent 工具调用、容错策略、设备控制链路治理。
```

## 示例 3：信息不足时的保留判断

### 输入

```markdown
岗位名称：Java后端工程师（AI Agent + IoT）

候选人信息：
- 简历摘要：5 年 Java 后端，做过一些 IoT 项目，也会用 AI 工具
```

### 输出示例

```markdown
## 任务目标
- 本次输出：简历初筛

## 初筛判断
- 当前结论：保留判断，不建议直接做录用或淘汰决定。

## 已知匹配点
- 具备 Java 后端基础。
- 简历中提到 IoT 项目和 AI 工具使用，方向相关。

## 风险与待验证项
- 缺少具体项目链路、技术栈、职责范围、结果指标。
- 无法确认是否做过 Agent 平台、设备控制链路、线上 SLA 保障。
- AI 工具使用深度不明，无法判断是否达到岗位硬性要求。

## 下一步建议
- 先补充项目经历、核心职责、技术栈和量化结果，再做评分卡。

## 待补充信息
- 负责过哪些核心模块
- 是否有设备接入或控制链路经验
- 是否做过 Agent 工具调用或编排
- AI 编程工具的具体使用场景与成果
```
